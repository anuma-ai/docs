# Backups

The SDK includes a backup system that syncs conversations to cloud storage. It
supports three providers: Dropbox, Google Drive, and iCloud. Each provider works
independently — you can use one, two, or all three.

## What Gets Backed Up

Backups include conversation metadata: ID, title, project association, and
timestamps. Each conversation is stored as an encrypted JSON file.

Messages and file attachments are not included in backups.

## Providers

### Dropbox

Conversations are stored in `/ai-chat-app/conversations/` as individual JSON
files. Requires a Dropbox App Key for OAuth authentication.

### Google Drive

Creates an `ai-chat-app` folder with a `conversations` subfolder. Each
conversation is a JSON file. Requires a Google OAuth Client ID with the
`drive.file` scope.

### iCloud

Uses Apple CloudKit to store conversations as records. Requires a CloudKit API
token and container identifier.

## How It Works

1. **Connect** — User authenticates with their chosen provider via OAuth
2. **Backup** — Conversations are exported, encrypted, and uploaded
3. **Restore** — Files are downloaded, decrypted, and imported locally

The backup system compares timestamps before uploading. If a conversation hasn't
changed since the last backup, it's skipped. This makes incremental backups
fast.

## Security

OAuth tokens are encrypted using the wallet address as the key and stored in
localStorage. Without a wallet address, tokens are stored temporarily in
sessionStorage and cleared when the page closes.

Conversations are encrypted before upload, so cloud providers cannot read the
content.
